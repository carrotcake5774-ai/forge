# Quality Gates — Multi-Agent Review

## 개요

각 STEP 완료 후 오케스트레이터(Opus)가 산출물을 검증한다.
미통과 시 1회 재실행, 2회 미통과 시 사용자 에스컬레이션.

---

## QG0: 에이전트 구성 적합성 (STEP 1 후)

| # | 검증 항목 | 기준 | 판정 방법 |
|---|----------|------|----------|
| QG0-1 | 에이전트 수 | 2~5명 (전문가 1~4 + Devil) | 카운트 |
| QG0-2 | 역할-도메인 매칭 | 각 역할이 문서의 핵심 도메인과 관련 | Opus 판단 |
| QG0-3 | Devil 포함 | Devil's Advocate 자동 추가 확인 | id=DEVIL 존재 |
| QG0-4 | ISS 부여 | 핵심 주장/제안에 ISS-NNN 부여 완료 | ISS 1개+ 존재 |
| QG0-5 | ISS-에이전트 배정 | 모든 ISS에 최소 1명 배정, DEVIL은 ALL | 매핑표 확인 |

### 미통과 처리

- QG0-1 위반: 에이전트 수 조정 후 재배정
- QG0-2 위반: 역할 재정의
- QG0-3~5: 누락 항목 추가

## QG1: 자기 설정 완전성 (STEP 1.5 후)

| # | 검증 항목 | 기준 | 판정 방법 |
|---|----------|------|----------|
| QG1-1 | 5개 항목 존재 | persona, review_focus, review_approach, needed_tools, severity_criteria 모두 포함 | 키 존재 확인 |
| QG1-2 | 항목 충실도 | 각 항목 200자 이상 (needed_tools 제외) | 길이 검사 |
| QG1-3 | 역할 일관성 | 자기 설정이 배정된 역할(role)과 일관 | Opus 판단 |
| QG1-4 | 차별화 | 에이전트 간 review_focus가 중복 < 50% | Opus 판단 |

### 미통과 처리

- QG1-1 위반: 누락 항목 명시하여 해당 에이전트만 재실행
- QG1-2 위반: "더 구체적으로 작성하세요" 지시로 재실행
- QG1-3~4: 역할 재확인 후 재실행

## QG2: 리뷰 커버리지 (STEP 2 후)

| # | 검증 항목 | 기준 | 판정 방법 |
|---|----------|------|----------|
| QG2-1 | ISS 커버리지 | 모든 ISS에 최소 1명이 언급 | ISS ID 카운트 |
| QG2-2 | 리뷰 길이 | 각 에이전트 리뷰 800자 이상 | 길이 검사 |
| QG2-3 | 프레임워크 항목 | 강점/약점/개선제안/확신도/전체평가 5개 모두 포함 | 섹션 존재 확인 |
| QG2-4 | 심각도 태그 | 약점 항목에 [Critical]/[Major]/[Minor] 태그 존재 | 태그 검색 |
| QG2-5 | Devil 역할 수행 | Devil이 최소 1건 가정 도전 또는 반론 포함 | 내용 확인 |

### 미통과 처리

- QG2-1: 누락 ISS 명시하여 해당 에이전트 재실행
- QG2-2: "더 심층적으로 분석하세요" 지시
- QG2-3~5: 누락 항목 명시하여 재실행

## QG3: 토론 참여도 (STEP 3 후)

| # | 검증 항목 | 기준 | 판정 방법 |
|---|----------|------|----------|
| QG3-1 | 교차 인용 | 각 에이전트가 최소 1명의 타 에이전트 리뷰를 직접 인용 | 인용 확인 |
| QG3-2 | 반론 존재 | 전체 R1에서 최소 1건의 반론([반론] 태그) 존재 | 태그 검색 |
| QG3-3 | Devil 가정 도전 | Devil이 합의 경향에 대해 최소 1건 비판적 질문/도전 | 내용 확인 |
| QG3-4 | ISS 상태 업데이트 | 언급된 모든 ISS에 상태 제안 존재 | 상태 태그 확인 |
| QG3-5 | 발언 형식 준수 | debate-protocol.md의 R1 발언 형식 준수 | 형식 확인 |

### 미통과 처리

- QG3-1~2: "다른 에이전트의 리뷰를 구체적으로 인용하여 반론하세요" 지시
- QG3-3: "합의된 견해에 대해 가정을 도전하세요" Devil에게 재지시
- QG3-4~5: 형식 보정 지시

## QG4: 수렴도 (STEP 4 후)

| # | 검증 항목 | 기준 | 판정 방법 |
|---|----------|------|----------|
| QG4-1 | ISS 전수 판정 | 모든 ISS에 RESOLVED 또는 ESCALATED 상태 | 상태 카운트 |
| QG4-2 | R1 반론 전수 대응 | R1의 모든 반론에 [수용] 또는 [유지-재반론] 응답 | 대응 카운트 |
| QG4-3 | 동일 근거 반복 금지 | [유지-재반론] 시 R1과 다른 새 논거 제시 | Opus 판단 |
| QG4-4 | ESCALATED 근거 | ESCALATED 항목에 양측 근거 모두 기록 | 내용 확인 |
| QG4-5 | Devil 블라인드스팟 | Devil의 블라인드스팟 점검 완료 | 내용 확인 |

### 미통과 처리

- QG4-1: 미판정 ISS 목록 제공하여 재실행
- QG4-2: 미응답 반론 목록 제공하여 재실행
- QG4-3: "새로운 논거를 제시하거나 수용하세요" 지시
- QG4-4~5: 누락 항목 보완 지시

### R3 트리거 판정

QG4 통과 후:
- RESOLVED 비율 >= 50% -> STEP 5 진행
- RESOLVED 비율 < 50% -> R3 실행 (Opus가 A안/B안 축소 제시)

| # | 검증 항목 | 기준 | 판정 방법 |
|---|----------|------|----------|
| QG5-1 | 9개 섹션 존재 | 문서개요/참여에이전트/ISS추적표/합의사항/조건부합의/미합의ESCALATED/블라인드스팟/종합평가/권고액션 | 섹션 카운트 |
| QG5-2 | ISS 전수 추적 | 모든 ISS가 추적표에 포함, 최종 상태 기록 | ISS 카운트 |

## 공통 규칙

### 재실행 정책

1. **1회 재실행**: 미통과 항목을 명시하여 해당 STEP만 재실행
2. **2회 미통과**: 사용자에게 에스컬레이션 — 미통과 항목과 산출물을 함께 표시하여 판단 요청
3. **재실행 시 이전 산출물 전달**: "이전 결과에서 아래 항목이 미충족입니다. 보완하세요:" 형식

### 검증 순서

```
STEP 0 (Input Parse) — QG 없음 (Opus 직접)
STEP 1 (Role Assignment) -> QG0
STEP 1.5 (Self-Config) -> QG1
STEP 2 (Independent Review) -> QG2
STEP 3 (R1 Debate) -> QG3
STEP 4 (R2 Debate) -> QG4 + R3 트리거 판정
STEP 5 (Critical Synthesis) -> QG5
STEP 6 (Final Report) — QG 없음 (Opus 직접)
```

### 검증 로그 형식

```markdown
### QG{n} 검증 결과
- 통과: QG{n}-1, QG{n}-2, ...
- 미통과: QG{n}-3 (사유: ...)
- 판정: PASS / FAIL (재실행 {m}회차)
```
